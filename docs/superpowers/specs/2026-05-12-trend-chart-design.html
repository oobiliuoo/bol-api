请求趋势图设计文档

日期：2026-05-12 | 状态：已批准 | bol-api

1. 概述

在管理后台概览页增加 Canvas 折线趋势图，按模型分色展示请求数/Tokens/费用随时间变化趋势。位于现有"模型用量统计"区域下方，与模型统计共用同一组时间周期按钮。

2. 后端

2.1 新增接口

GET /stats/trend?hours=168

参数

参数类型默认说明
hoursint168统计小时数（5/24/168/720）

颗粒度规则

hours桶大小说明
5小时最近5小时，每小时一个桶
24小时最近24小时，每小时一个桶
168天最近7天，每天一个桶
720天最近30天，每天一个桶

响应格式

{
  "granularity": "hour",  // "hour" | "day"
  "hours": 168,
  "series": [
    {
      "model": "gpt-4o",
      "data": [
        {"time": "2026-05-11T14:00:00Z", "requests": 10, "tokens": 500, "cost": 0.005},
        ...
      ]
    },
    ...
  ]
}

2.2 实现文件

app/db/crud.py — 新增函数

get_trend_data(session, hours) → dict

  • 根据 hours 决定颗粒度（≤24 → hour，≥168 → day）
  • SQLite 使用 strftime 按时间桶分组
  • 按 time_bucket, model 分组聚合：COUNT(*), SUM(request_tokens), SUM(response_tokens), SUM(cost)
  • 结果按 time_bucket 排序
  • 将扁平化的查询结果重构为按 model 分组的 series 数组
  • Top 8 模型单独成系列，其余合并为 model="其他"

app/routers/stats.py — 新增端点

GET /stats/trend

  • 验证管理员身份
  • 调用 get_trend_data()
  • 返回序列化结果

app/stats/models.py — 新增模型

  • TrendDataPoint: time, requests, tokens, cost
  • TrendSeries: model, data: list[TrendDataPoint]
  • TrendResponse: granularity, hours, series: list[TrendSeries]

3. 前端

3.1 Canvas 折线图（从零实现）

不引入 Chart.js 等外部库，用原生 Canvas 绘制。

图表结构

  • 画布区域：padding 上下左右留边距用于轴标签
  • Y 轴：自动计算最大值，绘制 4-5 条水平网格线及刻度标签
  • X 轴：按颗粒度显示时间标签（小时/天），间隔自动适配避免重叠
  • 数据线：每个模型一条折线，按 [x(time), y(value)] 坐标绘制
  • 面积填充：每条线下方半透明区域填充
  • 图例：模型名 + 色块，点击切换显示/隐藏对应线条
  • Tooltip：鼠标在图表区移动时，显示最近数据点的各模型数值

颜色方案

沿用现有模型柱状图的 10 色数组，按模型出现顺序循环分配：

const colors = ['#00d9ff', '#7b61ff', '#ff00aa', '#ff8c00',
                '#00ff88', '#ff3355', '#00aa88', '#aa00ff',
                '#88ff00', '#ff0088'];

交互

  • 指标切换：三个按钮"请求数 / Tokens / 费用"点击切换 Y 轴数据字段
  • 图例切换：点击图例项切换对应模型线条的可见性
  • 悬停提示：鼠标在图表上移动时，显示十字准线和数据点提示

5. 验证方式

  1. 启动服务器，打开管理后台
  2. 概览页在模型统计下方能看到"请求趋势"区域
  3. 默认显示请求数折线图，多条模型彩色线条
  4. 切换"Tokens / 费用"按钮，图形正确更新
  5. 切换"5时/日/周/月"时间按钮，图形正确更新（小时桶 vs 天桶）
  6. 点击图例项，对应线条显示/隐藏
  7. 鼠标悬停显示 tooltip
